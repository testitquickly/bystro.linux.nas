====== Про ssh ======

Secure shell (SSH) — наш основной способ коннекта на удалённый компьютер в консоли. То есть, подразумевается подключение через ту же консоль, но «с другим уровнем обеспечения безопасности».

В современных дистрибутивах этот сервис включен по-умолчанию (OpenSSH), его надо лишь дополнительно обезопасить и настроить, если в этом есть однозначная необходимость.

Ничто не сравнится со своим, личным, сделанным своими личными руками личным ssh-ключом. Гарантируется глубочайшее ложное ощущение полной безопасности.

===== Формат ssh-ключа для OMV =====

OMV принимает public key только двух форматов: 
1. OpenSSH (шифрование rsa, стандарт для утилиты ssh-keygen)
2. RFC 4716 (см. описание формата https://www.rfc-editor.org/rfc/rfc4716 )

Полностью отпадают ключи с крутыми современными алгоритмами вроде ECDSA или ed25519. С одной стороны, увы, с другой стороны — да и незачем.

Есть смысл сделать отдельный ключ для подключения к NAS через OMV — [[SSH:Сделать личный ssh-ключ]]

===== Софт для подключения по ssh =====

Консоль.

В мире виндовс привыкаешь к приложению ’PuTTY’, это в принципе та же консоль, но с расширенными настройками, и подключаться через неё можно не только по ssh. 

В Linux это ’PuTTY’ тоже есть:

''sudo aptitude install putty && putty &''
